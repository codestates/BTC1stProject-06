# BTC1stProject-06

## ⚡ 커밋 메시지의 형식
- 커밋 메시지의 각 줄은 100자 넘기지 않도록 작성
- 커밋 메시지 헤더 (Commit Message Header)
- type(scope): short summary

#### type에 들어갈 수 있는 항목들
- feat : 새로운 기능 추가
- fix : 버그 수정
- docs : 문서 관련
- style : 스타일 변경 (포매팅 수정, 들여쓰기 추가, …)
- refactor : 코드 리팩토링
- test : 테스트 관련 코드
- build : 빌드 관련 파일 수정
- ci : CI 설정 파일 수정
- perf : 성능 개선
- chore : 그 외 자잘한 수정
